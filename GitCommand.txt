git --version
git config --list
git config --global user.name "git hub user name"
git config --global user.email "git hub email id"

To clone existing repo
git clone <repositiry link>
git status
git add <file name or .>
git commit -m "message" <file name or .>
git push origin main

To push new repo created in system to remote repo(after creating a new repo in git hub and updating folder in system to git repo with git init command)
git remote add origin <repo link>
git remote -v (to verify remote)
git branch (to check branch)
git branch -M main (to rename branch)
git push origin main

branch commands
git branch (to check branch)
git branch -M main (to rename branch)
git checkout <branch name> (to navigate)
git checkout -b <new branch name> (new branch name)
git branch -d <branch name> (delete a branch)

merging code
way1
git diff <branch name> (to compare commits, branches, files and more)
git merge <branch name> (to merge 2 branches)
way2
Create a PR(pull request)

pull command
git pull origin main

Undoing changes
case1: staged changes
git reset <file name> (for specific file)
git reset (for multiple files)
case2: commited changes (for one commit)
git reset HEAD~1
case:3 commited changes (for many commits)
git reset <commit hash>
git reset --hard <commit hash>

git log (to check commit log)

fork (fork is a new repositiry that shares code and visibility settings with the original "upstream" repositiry)
